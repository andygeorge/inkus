## Project Overview

Inkus is an opinionated local Kubernetes cluster running on Incus VMs. The full stack is brought up by a single command. Network topology and size are configurable via `config.env`.

## Commands

```bash
make up       # install prereqs + provision VMs + configure k8s + print kubeconfig
make down     # terraform destroy + clean generated files
make status   # kubectl get nodes (uses ./kubeconfig)
make prereqs  # install host prerequisites only
make clean    # destroy cluster + remove all local state (.terraform, .ssh, etc.)
```

Users should never need to run terraform or ansible directly.

## Stack

- **terraform + dmacvicar/Incus provider** — VM provisioning (pool, network, volumes, cloud-init, domains)
- **ansible** — node configuration (containerd, kubelet, kubeadm, sysctl, kernel modules)
- **kubeadm + flannel** — k8s cluster bootstrap and CNI
- **cloud-init** — VM first-boot config (user, SSH key, static IP, hostname)

## Key Design Decisions

- **Static IPs via cloud-init** — DHCP is disabled on the Incus network. IPs are assigned in cloud-init network-config, derived from `config.env` values. This makes the API server address predictable for kubeadm.
- **SSH keypair per project** — generated in `.ssh/inkus` on first `make up`, gitignored. Used by both ansible and the SSH wait loop.
- **terraform.auto.tfvars** — generated by `up.sh` from `config.env` so terraform and ansible share the same config source. Gitignored.
- **Idempotency** — prereqs check before install, terraform is naturally idempotent, ansible roles use `creates:`/`stat` guards on kubeadm init/join, flannel checks for existing daemonset.
- **`config.env` is the only file users edit** — all terraform vars and ansible inventory vars are derived from it.
